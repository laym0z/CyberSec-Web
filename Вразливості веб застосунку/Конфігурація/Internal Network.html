Для того аби тестувати безпеку серверу, який знаходиться на віртуальній машині, я використовував Kali Linux яка теж знаходиться на віртуальній машині. Але обоє знаходилися в NAT мережі і тому не могли контактувати між собою простим шляхом. Тому я вирішив створити Internal Network (внутрішню мережу) між цими віртуальними машинами.

Для початку на обох машин варто змінити адаптер і дати СХОЖЕ ім'я:

Але оскільки DHCP по стандарту немає довелося роздати обом по статичній IP адресі:

Server

sudo vim /etc/netplan/01-intnet.yaml

network:
  version: 2
  ethernets:
    enp0s3:
      addresses:
        - 192.168.100.10/24


sudo netplan apply

Kali

sudo vim /etc/network/interfaces

auto eth0
iface eth0 inet static
    address 192.168.100.20
    netmask 255.255.255.0

sudo systemctl restart networking

Перевірити роботу мережі можна простим пінгуванням.

Але важливо зазначити, що оскільки це лише внутрішня мережа - доступу в Інтернет машини не матимуть!

Також в моєму випадку треба додати в firewall серверу правило на дозвіл підключення від IP Kali:

sudo ufw insert 3 allow from 192.168.100.20 to any port 8000 proto tcp
